🎫 Ticket Configurations

🎯 Email Assignment

Auto-assign incoming email tickets to agents.

Tickets will be manually assigned.

⏱ Email SLA Targets

Define response time goals for email tickets. Breaches are flagged.

🧠 Email Logic Rules

What happens when a customer replies on a resolved email ticket.

🔗 Email Threading

Control when a reply to an existing email creates a new ticket vs reopening the original.

How threading works:
Nexus checks for a matching ticket using: Gmail Thread ID → In-Reply-To header → Subject + Email match.
If a match is found, the behaviour above applies. If no match, a new ticket is always created.

🎯 Chat Assignment

Auto-assign incoming live chat conversations.

Chats will be manually assigned.

⏱ Chat SLA Targets

Response time goals for live chat conversations.

🧠 Chat Logic Rules

Behaviour when a customer messages a resolved chat ticket.

🔗 Chat Threading

Control what happens when the same customer starts a new chat session.

How chat threading works:
When a customer submits the pre-chat form, Nexus checks their email address for an existing ticket matching the rules above. If a match is found, it continues or reopens that ticket instead of creating a new one.

🎯 Social Assignment

Auto-assign incoming social media tickets.

Social tickets will be manually assigned.

🧠 Social Logic Rules

Behaviour when a customer comments on a resolved social ticket.

🔗 Social Threading

Control how replies and comments on social tickets are handled.

How social threading works:
Social tickets are matched by platform username or profile ID. Replies to the same post/comment chain are grouped based on your settings above.

🔑 Nexus REST API

                
Your tech team can use these credentials to read/write tickets, messages, and customer data programmatically.

                

                  

                    
                    
                  

                

                

                  

                    
                    
                    
                  

                  
Pass as Authorization: Bearer <key> header

                

                

                  Quick examples:

                  GET /api/convs — list all tickets

                  GET /api/convs?ch=email&st=open — filter

                  POST /api/convs/:id/msgs — send a message

                  PATCH /api/convs/:id — update status/agent

                  GET /api/customers — customer list

                  GET /api/analytics — dashboard metrics

🔗 Outbound Webhooks

                
Nexus will POST a JSON payload to your URL whenever these events fire. Use this to connect your bot, CRM, or any external system.

                

                  

                    
📥 new_ticket

                    
Fires when a new ticket is created from any channel

                  

                  

                    
💬 new_message

                    
Fires when a customer sends a new message

                  

                  

                    
✅ ticket_resolved

                    
Fires when a ticket status changes to resolved

                  

                  

                    
👤 agent_assigned

                    
Fires when a ticket is assigned to an agent

                  

                  

                    
⭐ csat_received

                    
Fires when a customer submits a CSAT rating

                  

                  

                    
🔴 sla_breach

                    
Fires when a ticket breaches its SLA target

                  

                


                

                  

                    
                    
                  

                

                

                  
                

                

                  

                    
                    
                    
                    
                    
                    
                  

                

                
                

                  Payload format (JSON):

                  { "event": "new_ticket", "data": { ticket }, "timestamp": "ISO" }

                  Signature header: X-Nexus-Signature: sha256=<hmac>
                

              

            


            
            

              
📨 Inbound API — Bot sends messages INTO Nexus

              
Your bot can create tickets and send messages into Nexus via these endpoints:

              

                Create a chat ticket:

                POST /api/widget/start

                Body: {"name": "Customer Name", "email": "[email protected]"}


                Send a message into a ticket:

                POST /api/widget/send

                Body: {"convId": "<id from above>", "content": "Message text", "name": "Bot Name"}


                Read all messages in a ticket:

                GET /api/widget/msgs/:convId


                Resolve a ticket:

                PATCH /api/convs/:id  Body: {"status": "resolved"}

🤖 Bot Integration

                
Connect your external bot. Nexus will forward customer messages to your bot and display its replies as agent messages.

                
Enable Bot Forwarding

                

                  
                

                

                  
                

                

                  
                

                

                  

                    
                    
                    
                  

                

                
                

                  How it works:

                  1. Customer sends a message → Nexus receives it

                  2. Nexus POSTs to your bot URL: {"message": "...", "convId": "...", "channel": "..."}

                  3. Your bot calls POST /api/widget/send with the reply

                  4. Reply appears in Nexus as a bot message
